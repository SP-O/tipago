# 티카투카 AI 어시스턴트 (최적 수 추천 솔버) — 설계 문서

작성일: 2026-06-24

## 1. 개요 / 목적

티카투카(주사위 대전 게임)를 플레이하는 동안, **현재 판 상태를 입력하면 "이길 확률이 가장 높은 수"와 그 승률을 알려주는** 웹 기반 솔버. "알파고처럼" 최선의 수로 이끌어주는 보조 도구.

- 대상 게임: PC/웹 주사위 게임(티카투카). 사용자는 AI 상대와 대전.
- 사용 형태: 게임 옆에 브라우저 창으로 띄워두고, 현재 판을 **수동 입력** → 추천 수 + 승률 확인.
- 배포: 정적 웹앱(예: Vercel). 링크 공유로 친구들도 사용 가능.

## 2. 게임 규칙 (엔진이 정확히 구현해야 할 사양)

### 2.1 보드 구조
- 두 명(나 / 상대). 각자 **3개 라인**, 각 라인은 **최대 3칸**.
- 라인은 **짝**으로 비교: 내 라인 i ↔ 상대 라인 i.
- 칸 상태: 비어있음 | { 값: 1~6, 실드: true/false }.

### 2.2 점수(라인 합) 계산
라인 안의 주사위를 값별로 묶어 합산:
- 같은 값 1개: `v`
- 같은 값 2개(더블): `3v` (주사위 2개분 + 1개분 보너스) — 예) 5,5 → 15
- 같은 값 3개(트리플): `5v` (주사위 3개분 + 2개분 보너스) — 예) 5,5,5 → 25
- 혼합 예) 3,3,5 → (3더블 9) + 5 = **14**
- 라인 합 = 각 묶음 기여분의 총합.
- **실드 주사위도 점수·더블·트리플에 일반 주사위와 동일하게 포함.** 실드는 "제거 면역" 기능만 추가.

### 2.3 턴 진행
- **선공은 랜덤.** 선공의 **첫 주사위는 실드**로 제공.
- 번갈아 진행. 한 턴:
  1. (선택) 밑장빼기 발동 가능 — 2.6 참고
  2. 주사위 1개를 굴림(랜덤 1~6)
  3. 빈칸이 있는 **내 라인 중 하나**에 배치
  4. 알까기 발동 시 → 보너스 주사위를 한 번 더 배치(2.4)
- **게임 종료:** 양쪽 9칸(총 18칸)이 모두 채워지면 끝.

### 2.4 알까기 (핵심 메커니즘)
- **발동 조건:** 내가 굴린 값 `V`를 **내 라인 i**의 빈칸에 두려 할 때, **상대 라인 i**(같은 라인)에 값 `V`인 **비(非)실드** 주사위가 1개 이상 있으면 발동. (다른 라인의 상대 주사위는 영향 없음.)
  - "내 필드에 빈칸 있을 때만 발동" = 트리거 주사위를 놓을 빈칸이 그 라인에 있어야 함.
- **효과(순서):**
  1. 내가 둔 주사위 `V`도 함께 사라짐(그 칸은 다시 비워짐).
  2. 상대 라인 i의 값 `V`인 **비실드** 주사위 전부 제거(실드는 안 날아감).
  3. **보너스 주사위 1개** 지급: 새로 굴린 **랜덤 값**, **실드** 속성.
  4. 보너스 주사위를 **아무 빈칸(내 필드 또는 상대 필드)**에 배치 — 이것이 "한 번 더 놓기".
- 보너스 주사위 배치는 **추가 알까기를 연쇄하지 않음.**
- 상대(시뮬레이션)도 동일하게 나에게 알까기 가능. 내 실드는 면역.

### 2.5 실드 주사위
- 출처: ① 알까기 보너스 주사위, ② 선공의 첫 주사위.
- 알까기로 제거되지 않음(면역). 그 외에는 일반 주사위와 완전히 동일(점수 동일).

### 2.6 밑장빼기 (게임당 1회, 반응형)
- 주사위를 굴려 값 `V`를 **본 뒤**, 마음에 들지 않으면 발동 가능.
- 발동 시 주사위를 한 번 더 굴림 → 새 값 `V2`. **`V2`는 `V`와 절대 같지 않음** (1~6 중 `V` 제외, 균등).
- `V` 와 `V2` 중 **하나를 골라** 그 턴에 배치.
- **게임당 1회만.** 나는 항상 보유 가정. 상대는 **설정 토글**로 사용 여부 지정(상대 등급 대응).

### 2.7 승패 판정
- 라인별로 내 합 vs 상대 합 비교: 큰 쪽이 그 라인 승. **같으면 무승부(둘 다 못 가짐).**
- 이긴 라인 수가 더 많은 쪽이 게임 승. **같으면 전체 무승부.**
- (예: 1:1 + 1라인 동점 → 1 대 1 → 전체 무승부. ※ 실제로 본 적 없는 상황이라 일단 무승부로 가정.)

## 3. 전략 원칙 (평가 함수·롤아웃 정책·검증 테스트의 기준)

이 게임은 "눈이 큰 주사위를 쌓는" 게임이 아니라 **상대 제거로 라인을 뒤집는** 게임. 솔버가 반드시 반영해야 할 원칙:

1. **상대 중복 제거 최우선.** 상대 라인에 같은 눈 2개 이상(특히 4·5·6)이면 거의 무조건 그 라인을 알까기로 친다(제거 = 점수격차 + 보너스 주사위 1회).
2. **라인을 3개로 막지 않기.** 라인이 3칸으로 꽉 차면 상대가 일방적 고점을 가질 수 있음. 기본 운영은 **3라인에 2개씩** 두며 제거각을 보는 것.
3. **총점이 아니라 2/3 라인 승리.** 이미 크게 이긴 라인 과투자 금지. 지는 라인(예 5:8)에 투자해 뒤집기(11:8)가 훨씬 가치 높음.
4. **실드 배치는 고점 중복 라인 잠그기.** 보너스(실드) 값이 높으면 내 이기는 라인의 핵심 숫자 중복으로 잠금. 낮으면 상대 라인 빈칸에 박아 라인을 죽이는(점유로 망치는) 용도.

## 4. 아키텍처 (3층 분리)

```
① UI 층      : 보드형 수동 입력 + 추천/승률 출력
② 게임 엔진   : 규칙 시뮬레이터(점수·알까기·실드·밑장빼기·합법수·승패·종료)
③ 솔버(두뇌)  : 하이브리드(MCTS + 후반 완전탐색), 최적 상대 가정, 승률 산출
```

- 엔진은 솔버와 독립적으로 단위 테스트 가능.
- 솔버는 엔진을 "블랙박스 시뮬레이터"로만 사용 → 두뇌를 나중에 더 강화해도 UI/엔진 불변.

## 5. 게임 엔진 상세

- **상태 모델:** 두 플레이어 × 3라인 × 3칸(값/실드), 각 플레이어의 밑장빼기 보유 여부, 현재 턴, 굴린 값.
- **합법 수 생성:** 굴린 값 `V`에 대해 빈칸 있는 내 라인들이 후보. 후보 라인이 알까기를 발동하면, 그 가지(branch)는 "보너스 주사위 배치 위치"까지 포함해 확장.
- **점수/승패/종료:** 2.2 / 2.7 / 2.3 규칙대로.
- **결정성:** 무작위 요소(주사위, 보너스, 밑장빼기 리롤)는 명시적 chance 노드로 분리해 솔버가 확률 가중 가능하도록 한다.
- **종료 안전장치:** 알까기가 칸을 다시 비우므로 이론상 장기화 가능 → 시뮬레이션에 턴 상한(safety cap)을 두고, 도달 시 현재 보드로 승패 평가(무한루프 방지).
- **엣지:** 보너스 주사위를 놓을 빈칸이 전혀 없으면(희귀) 보너스 소멸 처리.

## 6. 솔버 상세

### 6.1 모델
- 게임 트리: **내 결정 노드(max) / 상대 결정 노드(min) / 주사위 chance 노드(기댓값)**.
- **상대는 최적(강하게) 가정** → 표시 승률 = "양쪽 최선 + 주사위 무작위" 하에 내가 이길 확률(보수적·안전).
  - 참고: 실제 게임 AI는 알까기를 항상 쓴다고 함. 최적 상대는 이득일 때만 씀 → 실제 AI가 손해 보며 남발하면 우리 추천은 그만큼 안전 쪽. (추후 보정 여지.)

### 6.2 하이브리드 탐색
- **중반(빈칸 많음): 몬테카를로 / MCTS.** 양쪽 결정 노드를 UCT로 탐색, chance 노드는 샘플링. 끝까지 롤아웃하여 승/무/패 집계 → 승률. 롤아웃 정책에 **3장 전략 원칙** 반영.
- **후반(빈칸 적음, 임계치 이하): 완전탐색(기댓값 미니맥스).** 남은 경우의 수를 모두 전개해 **정확한 승률** 산출. 자동 전환.
- 주사위 확률 1~6 균등, **밑장빼기 리롤은 기존 값 제외** 반영.

### 6.3 출력
- 현재 굴린 주사위의 **모든 합법 배치**(+ 알까기 시 보너스 배치 위치)별 **승률**.
- 최고 승률 수를 ⭐추천(보드에서 하이라이트).
- **밑장빼기 추천:** 밑장빼기 보유를 상태에 포함하므로, 최적 탐색이 "지금 쓰는 것"을 택하면 권장으로 표시(예: "지금 4 그냥 58% vs 밑장빼기 63% → 권장"). 저장해 두는 가치도 자동 반영.

### 6.4 성능
- 게임 규모 작음(최대 18 주사위). 솔버는 **Web Worker**에서 실행해 UI 비차단. 수천 회 시뮬레이션을 1초 내외로 목표.

## 7. UI / UX

- **입력:** 보드형. 상대 필드(위)·내 필드(아래) 각 3라인×3칸. 칸 클릭→값(1~6), 재클릭→실드 토글(테두리색). "내가 굴린 주사위" 값 선택, "밑장빼기 남음" 체크, "상대 밑장빼기 사용" 설정 토글, "추천 계산" 버튼.
- **출력:** ⭐추천(위치+승률), 알까기면 보너스 배치 추천 동반, 밑장빼기 권장 여부, 나머지 선택지 승률 순위. 각 라인 합은 더블/트리플 반영해 자동 표시. 계산 중 스피너.

## 8. 기술 스택

- **순수 웹앱**: HTML + JavaScript(바닐라 또는 경량). 빌드 부담 최소화.
- **Web Worker**로 솔버 분리.
- 정적 호스팅(Vercel 등) 배포. 기존 프로젝트와 별개의 **독립 폴더**(예: `tikatuka-assistant/`)로 구성.

## 9. 테스트 전략

엔진이 틀리면 승률 전체가 틀어지므로 **엔진 단위 테스트 우선**.
- 점수: 5,5→15 / 5,5,5→25 / 3,3,5→14 / 전부 다른 값 단순합.
- 알까기: 같은 라인만 제거 / 실드 면역 / 내 주사위도 소멸 / 보너스 1개 지급·배치 / 다른 라인 불변.
- 밑장빼기: 리롤 값 ≠ 기존 값 / 게임당 1회 / 선택 반영.
- 승패: 라인별 합 비교, 동점 무승부, 1:1:동점 → 전체 무승부, 18칸 종료.
- 솔버: 손으로 풀 수 있는 후반 상황의 승률·추천 검증 + 3장 전략 원칙에 부합하는지(제거각 우선, 라인 과투자 회피, 지는 라인 뒤집기, 실드 잠금) 시나리오 테스트.

## 10. 추후/보정 항목 (v1 범위 밖, 기록만)

- 화면 캡처 자동 인식(반자동 입력) — v1은 수동 입력.
- 실제 게임 AI 관찰 기반 상대 모델 보정 / 상대 등급별 강도 설정.
- 1:1:동점 실제 판정 확인 시 규칙 갱신.
