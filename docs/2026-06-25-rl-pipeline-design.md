# 티카투카 자가학습(RL) 파이프라인 — 설계 문서

작성일: 2026-06-25

## 1. 목표

기존 규칙 엔진을 시뮬레이터로 삼아 **self-play 강화학습**으로 신경망(가치/정책)을 만들고, 이를 **MCTS의 평가 함수**로 써서 현재의 단순 그리디 몬테카를로보다 강한 추천을 제공한다. 학습은 사용자 PC에서 별도로 돌리고(대화 토큰 0), 결과 모델만 웹앱에 이식한다.

## 2. 확정된 결정

- **스택: 순수 JS (Node).** 기존 `src/state.js / scoring.js / rules.js` 엔진을 그대로 재사용(규칙 중복 구현 없음 → 학습/배포 규칙 불일치 위험 제거).
- **신경망: 작은 MLP를 직접 구현**(외부 ML 라이브러리 없음). forward/backprop/Adam을 손으로 짜고 **그래디언트 체크로 검증**. 같은 forward 코드를 브라우저에서도 사용 → 의존성 0, 모델 파일은 수십 KB~1MB 미만.
- **학습은 PC에서** `node train.js`로 실행, 체크포인트/모델 저장. 브라우저는 학습 안 함(추론만).

## 3. 단계적 인도 (위험 최소화)

- **Stage 1 — 학습 기반 검증:** 인코딩 + MLP(forward/backprop/Adam) + MCTS 골격. **그래디언트 체크 통과**, **작은 합성 데이터 과적합 테스트 통과**(학습 기계 자체가 옳음을 증명). 아직 self-play 없음.
- **Stage 2 — self-play 학습:** 자기 대국 → 기보 수집 → 가치(필요 시 정책) 학습 → 반복. **평가 게이트**: 새 모델이 현재 그리디-MC 솔버 및 직전 체크포인트를 상대로 승률 우위일 때만 채택.
- **Stage 3 — 브라우저 연결:** 학습된 `model.json`을 Worker가 로드, 신경망+MCTS로 추천/승률 산출. 기존 UI 그대로.

## 4. 구성 요소 (파일)

```
src/solver/
  encode.js     # 상태 → 특징 벡터(입력), 행동(action) 열거/마스킹
  net.js        # MLP forward (브라우저+학습 공용)
  nn-train.js   # 학습 전용: backprop, Adam, 손실 (Node에서만 사용)
  mcts.js       # 신경망 가이드 MCTS (주사위 chance 노드 + 알까기/밑장빼기 노드)
  model.json    # 학습된 가중치 (Stage 2 산출물)
train.js        # self-play 학습 루프 (PC에서 실행)
tests/
  encode.test.js, net.test.js (그래디언트 체크/과적합), mcts.test.js
```

## 5. 신경망 설계 (1차)

- **입력 특징(약 80~120차원):** 6개 라인(me 3 / opp 3) × 3칸 각각의 주사위 값(one-hot 1~6, 7번째=빈칸) + 실드 플래그; 각 라인 현재 합(정규화); 양측 밑장빼기 보유; 현재 의사결정 단계(turn-die / bonus / mitjang); 현재 다룰 주사위 값; 차례(me/opp).
- **본체:** Dense(→128) ReLU → Dense(→128) ReLU.
- **가치 head:** Dense(→1) + sigmoid = me 관점 승률. (핵심 — 약한 무작위 롤아웃을 대체)
- **정책 head(선택, Stage 2+):** 통합 행동 집합에 대한 softmax. 행동 집합:
  - 내 라인0/1/2에 현재 주사위 배치 (3)
  - 보너스 주사위를 me0..2 / opp0..2에 배치 (6)
  - 밑장빼기 사용 (1)
  → 총 10, 단계별 합법 행동만 마스킹. (1차에는 정책 head 없이 휴리스틱 prior로 시작 가능)

## 6. MCTS 설계

- 의사결정 노드(PUCT)와 **주사위 chance 노드(기댓값/샘플)**를 모두 다룸. 턴 구조(밑장빼기→배치→알까기→보너스)는 엔진 프리미티브(`resolveAlkkagi/placeDie/endTurn/setMitjang`)로 전개.
- 리프 평가 = **가치 신경망**(무작위 롤아웃 대체). prior = 정책 head 또는 (없으면) 기존 휴리스틱 `chooseScore`.
- me 노드는 승률 최대화, opp 노드는 최소화(현재 "상대 최적" 가정 유지). 시뮬 횟수는 설정값.

## 7. 학습 루프 (Stage 2, PC 실행)

1. 현재 모델로 MCTS self-play N게임 → (상태, MCTS 가치/방문분포, 최종 결과) 수집(리플레이 버퍼).
2. Adam으로 가치(MSE)+정책(교차엔트로피, 정책 head 있을 때) 학습.
3. 평가: 새 모델 vs 직전/그리디-MC, K게임. 승률 ≥ 임계치면 채택, 아니면 폐기.
4. 1~3 반복, 체크포인트 저장. 사용자가 적당히 강해지면 중단.

## 8. 성공 기준

- Stage 1: 그래디언트 체크 오차 < 1e-4, 작은 데이터셋 손실이 ~0으로 수렴(과적합 가능).
- Stage 2: 학습 모델이 현재 그리디-MC 솔버 상대로 분명한 승률 우위(예: ≥60%).
- Stage 3: 브라우저에서 모델 로드 + 추천/승률 정상 출력, 콘솔 에러 0.

## 9. 정직한 한계 (반드시 인지)

- **주사위 게임**이라 "절대 안 짐"은 불가능. 어시스턴트는 "승률 최대 수"를 제시할 뿐.
- **self-play로 강함 ≠ 실제 인게임 AI 예측 정확도.** 자가학습은 최적 자신을 상대로 강해지는 것. 실제 게임 AI에 대한 승률 보정이 필요하면 별도의 "실제 AI 모델링"(Option C)이 필요.

## 10. 추후

- 정책 head 추가로 탐색 효율↑.
- 실제 AI 데이터로 상대 모델 보정(Option C)과 결합.
- 모델 양자화로 추가 경량화.
