# Step 2 — 제대로 된 AlphaZero 재설계 설계 문서

작성일: 2026-06-25

## 0. 가장 중요한 원칙 — 합격 기준(Acceptance Gate)

지난 1차 NN은 "가치망 1수 앞보기"라 out-of-distribution 상황에서 오판(고점을 상대 필드에 추천)했다. 같은 실수를 막기 위해, **새 NN은 다음을 모두 통과해야만 브라우저 기본 엔진으로 채택**한다. 못 넘으면 **규칙 기반 솔버(정밀 솔버)를 그대로 유지**한다(더 나쁜 NN을 절대 출시하지 않는다).

1. **head-to-head**: 새 NN(MCTS 포함)이 규칙 기반 솔버를 상대로 N=200판 이상에서 **승률 ≥ 55%**.
2. **프로 전략 회귀 테스트 전부 통과**(`tests/solver-quality.test.js` + 추가분).
3. **블런더 테스트 통과**: 보너스 고점(6) 상대 필드 추천 같은 명백한 악수 금지(전용 테스트).

→ 이 게이트가 "헛수고 방지" 장치다. 결과적으로 NN이 못 이기면, 그 사실 자체가 결론(이 작은 게임은 규칙 기반이 천장).

## 1. 왜 1차 NN이 실패했나 (재발 방지)

- **가치망 1수 앞보기**: 리프 NN값을 그대로 신뢰 → 안 가본 수의 엉터리 값을 집음.
- **정책망 없음**: 모든 후보를 동일하게 평가 → 나쁜 수도 후보로.
- **얕은 self-play 정책**: 데이터 품질 낮음.

## 2. 핵심 변경 (제대로 된 AlphaZero)

1. **정책+가치 신경망**: `net.js`에 정책 head 추가(공통 몸통 → 정책 logits + 가치). 가치는 기존대로 me 관점 승률.
2. **행동 공간(action space) 정의**: 통합 10개 — 내 라인0/1/2 배치(3) + 보너스 me0..2/opp0..2(6) + 밑장빼기 사용(1). 단계별 합법 행동만 마스킹.
3. **진짜 MCTS**: PUCT 트리 탐색(1수 ❌ → 시뮬 수백 회). 주사위는 chance 노드(기댓값/샘플). 리프는 가치망. prior는 정책망. **루트 Dirichlet 노이즈**로 탐험. → 트리 탐색이 엉터리 리프값을 교정.
4. **AZ 학습 루프**: self-play를 MCTS로 진행, (상태, MCTS 방문분포=정책타깃, 최종결과=가치타깃) 수집 → 학습 → 반복. 온도 점감, 리플레이 버퍼.
5. **평가 게이트**: 매 N반복마다 규칙 기반 솔버와 head-to-head, 0장 기준 충족 시에만 model.json 갱신.

## 3. 파일

```
src/solver/
  net.js        # (확장) 정책 head 추가
  az-encode.js  # 상태+단계+굴린주사위 → 입력, 행동 마스킹
  mcts.js       # PUCT 트리 + chance 노드 + Dirichlet
  az-train.js   # AZ self-play 학습(정책+가치 타깃)
  az-recommend.js # MCTS 기반 추천(브라우저)
train-az.js     # PC 실행 학습 스크립트
tests/
  mcts.test.js, az-encode.test.js, az-gate.test.js(합격 기준)
```

## 4. 단계적 인도

- **2a**: 정책+가치 net + 행동 인코딩/마스킹 + 그래디언트 체크.
- **2b**: PUCT MCTS(chance 노드) + 단위 테스트(결정 상황에서 올바른 수).
- **2c**: AZ self-play 학습 스크립트(PC 실행) + head-to-head 평가.
- **2d**: 합격 게이트 통과 시에만 브라우저 연결(useNN 기본화). 못 넘으면 classic 유지.

## 5. 정직한 리스크

- 작은 게임이라 규칙 기반(기댓값 미니맥스+MC)이 이미 거의 최적 → NN이 못 이길 수도 있음. **그 경우 classic 유지가 정답**(게이트가 보장).
- 큰 작업량. 단계별로 검증하며 진행.
